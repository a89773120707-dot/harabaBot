# Вопросы для обсуждения — Сессия 2

## 🔴 Критичные (решить в начале)

### 1. Парсер региона — расширить паттерны
**Проблема:** Регион "Тверская область, посёлок городского типа Кесова Гора" не распознаётся → `region: unknown`.
**Текущий парсер** ищет только конкретные города (Ярославль, Москва, Тверь...).
**Что нужно:** Добавить:
- Области целиком ("Тверская область", "Ярославская обл.")
- Посёлки/районы с привязкой к области
- Город + область через запятую

### 2. Регион из URL объявления (fallback)
**Проблема:** Если на mobile detail странице регион не найден в raw_text.
**Идея:** Парсить регион из URL авто.ру (auto.ru/moskovskaya_oblast/cars/...) или из resolved_urls.

## 🟡 Важные

### 3. Telegram — не отправлять пачкой
**Проблема:** Когда приходят 3+ карточки одновременно — теряется контекст reply.
**Идея:** Отправлять по 1 карточке с задержкой 2-3 сек, или группировать с заголовком.

### 4. hold-карточки: что делать с region unknown
**Решение принято:** Отправлять в Telegram с предупреждением ⚠️ "Регион не распознан — проверить вручную".
**Вопрос:** Нужно ли при scoring учитывать unknown регион как warning или пропускать?

### 5. Telegram-карточка: видна ли вся информация
**Проверено:** 1 тестовая карточка, ссылка работает, кнопки работают.
**Нужно:** Проверить на 3-5 разных машинах — не обрезается ли текст (Telegram лимит 4096 символов).

## 📝 Принятые решения (зафиксировать)

| Решение | Почему |
|---------|--------|
| Скоринг НЕ фильтрует, а объясняет | Цель v1 — сбор реакций, не идеальный автофильтр |
| Отправлять ВСЕ карточки из конфига (кроме hard-stop) | Пользователь сам размечает рынок |
| hard-stop: manual / not_awd / wrong_region / legal_restrictions / duplicate | Критичные проблемы |
| hold-карточки отправлять с ⚠️ предупреждением | Не блокировать, но пометить |
| Mobile URL для ссылок в Telegram | Desktop ссылки не открываются на телефоне |
| reply_to_message_id для кнопок и комментариев | Чтобы было видно к какой карточке относится реакция |
| feedback.db не удалять при перезапусках | Реакции должны накапливаться |
| Один экземпляр бота (409 Conflict) | Только один polling instance |
